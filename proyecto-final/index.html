🚀 Proyecto Final: Desarrollo Web Integrador

En esta tercera unidad darás el paso más importante del curso: desarrollar un proyecto web completo aplicando todo lo aprendido sobre HTML, CSS, diseño responsivo y JavaScript.

En esta ocasión asumirás el rol de un técnico programador web, cuya tarea es tomar un RFC (Request for Comments) —un documento técnico que define los requerimientos del proyecto— y convertirlo en un sitio web funcional 💻.

Tu trabajo será analizar, comprender sus especificaciones y llevarlo a la práctica, escribiendo el código necesario para cumplir con los objetivos del proyecto. Así trabajarás como lo hacen los desarrolladores en entornos reales: interpretando requerimientos, resolviendo problemas y entregando soluciones efectivas ✨.

Objetivos de aprendizaje

  • 📋 Analizar e interpretar un RFC (Request for Comments) para identificar los requerimientos técnicos y funcionales de un proyecto web.
  • 🔧 Implementar la estructura, diseño y funcionalidad de un sitio web utilizando HTML, CSS y JavaScript de manera integrada.
  • ✅ Aplicar buenas prácticas de desarrollo web para garantizar la funcionalidad, responsividad y calidad del proyecto final.
  • 💻 Desarrollar un proyecto web completo que integre todos los conceptos aprendidos durante el curso en un contexto profesional real.

¿Qué es un RFC?

Un RFC (Request for Comments) es un documento técnico que describe cómo debe funcionar un sistema, aplicación o componente. En el desarrollo web, el RFC sirve como guía de trabajo para los programadores: explica qué se necesita construir, cómo debe comportarse el sistema y qué reglas o requisitos debe cumplir.

En otras palabras, el RFC traduce las ideas del proyecto en instrucciones técnicas claras. El programador no inventa el diseño ni los objetivos: su tarea es leer, analizar e implementar lo que el RFC indica.

📄 Análisis del RFC del proyecto

A continuación, encontrarás el RFC (Request for Comments) del proyecto que vas a desarrollar. Léelo con atención, identifica qué partes del trabajo te corresponden implementar y qué requisitos debes cumplir.

Recuerda: tu función como técnico programador es tomar este documento y convertirlo en un sitio web funcional.

💡 Consejo: Navega por las páginas del documento usando los controles inferiores o arrastrando las páginas. Lee con atención cada sección del RFC para identificar todos los requerimientos del proyecto.

Actividades del proyecto

A continuación encontrarás las tres actividades que debes completar para desarrollar exitosamente tu proyecto final. Sigue cada paso cuidadosamente y asegúrate de cumplir con todos los requerimientos del RFC.

📋 Actividad 1: Análisis del RFC

Lee detenidamente el RFC entregado por el docente e identifica qué partes del proyecto debes implementar (estructura, diseño y comportamiento). Realiza una lista de los principales requerimientos o funciones que debe cumplir tu sitio web.

💡 Consejo: Toma notas de cada sección del RFC y organiza los requerimientos por categorías: HTML (estructura), CSS (diseño) y JavaScript (funcionalidad).

💻 Actividad 2: Desarrollo del proyecto web

Crea la estructura HTML siguiendo buenas prácticas semánticas. Aplica estilos con CSS, utilizando Flexbox y Grid para el diseño y asegurando que el sitio sea responsivo. Implementa con JavaScript las funcionalidades indicadas (eventos, manipulación del DOM y validaciones).

💡 Consejo: Desarrolla el proyecto por etapas: primero la estructura HTML, luego los estilos CSS y finalmente la funcionalidad con JavaScript. Prueba cada parte antes de continuar.

✅ Actividad 3: Pruebas y validación

Prueba el sitio web en diferentes navegadores y tamaños de pantalla. Corrige errores y asegúrate de que todas las funcionalidades respondan según lo indicado en el RFC.

💡 Consejo: Usa las herramientas de desarrollo del navegador (F12) para probar en diferentes dispositivos. Verifica que el diseño sea responsivo y que todas las funciones JavaScript operen correctamente.

📌 Nota importante: Recuerda que el proyecto debe cumplir con todos los requerimientos especificados en el RFC. Consulta con tu docente si tienes dudas sobre alguna especificación.
