Definición de API

Imagina que pides una hamburguesa en un restaurante. Tú no entras a la cocina, no decides qué ingredientes se usan ni cómo se prepara todo. Solo haces el pedido de una forma clara, esperas, y te entregan el resultado sin sabes que ocurre detrás en el proceso de preparación de la orden.

En el mundo web pasa algo parecido. Muchas veces una página o una aplicación necesita información que no tiene guardada, o necesita que “algo suceda” en otro lugar: iniciar sesión, guardar un comentario, ver un listado de productos o consultar el clima. Para eso existe una API.

Definición de API

¿Qué es un API?

Una API (Application Programming Interface) es un conjunto de reglas para que un programa pueda usar datos o servicios de otro programa. Dicho de forma simple: es una forma organizada de pedir y recibir información.

En Backend, una API suele permitir acciones muy comunes en la vida real de una aplicación: consultar información, enviar información nueva, actualizar datos que ya existen o eliminar datos cuando ya no se necesitan. Gracias a eso, una página deja de ser solo “bonita” y se vuelve útil: puede mostrar datos de una base de datos, guardar formularios, crear usuarios o actualizar perfiles.

Las APIs también ayudan a que una aplicación esté mejor organizada. El Frontend se encarga de lo que el usuario ve y usa. El Backend se encarga de la lógica y los datos. Separar esas responsabilidades hace que sea más fácil mejorar la app, corregir errores y, más adelante, conectar la misma API a diferentes interfaces (una web, una app móvil o incluso un sistema de escritorio).

💡Ejemplo

Un ejemplo sencillo es el clima. Tu app pide “Dame el clima de Montería” y el servidor responde “28°C, nublado”. Tu app no calcula el clima; solo lo muestra. La API es la que permite que esa comunicación sea clara.

Para no enredarse, conviene recordar cuatro palabras: cliente, servidor, petición y respuesta.

  1. 1. El cliente es quien pide (por ejemplo, el navegador).
  2. 2. El servidor es quien responde (tu Backend).
  3. 3. La petición es el mensaje que envías para pedir algo.
  4. 4. La respuesta es lo que recibes de vuelta.
