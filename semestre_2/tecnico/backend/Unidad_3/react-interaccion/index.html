Interacción y actualización de datos: estado, eventos, hooks básicos

Imagina que estás usando Instagram. Cuando le das "like" a una foto, el corazón se pone rojo instantáneamente. Cuando escribes un comentario, ves cada letra aparecer en tiempo real. Cuando actualizas el feed, aparece un "Cargando..." mientras trae nuevas publicaciones.

¿Cómo funciona esa magia? 🪄

La respuesta es: estado y eventos. El estado es la "memoria" de tu componente (como el número de likes), y los eventos son las acciones del usuario (como hacer clic en el corazón).

🚦 Analogía del semáforo:

  • El semáforo tiene un estado: rojo, amarillo o verde
  • Cada cierto tiempo ocurre un evento: cambiar de color
  • Cuando cambia el estado, la luz que ves se actualiza automáticamente

React funciona igual: cuando el estado cambia, la pantalla se actualiza sola. Tú no tienes que decirle "ahora cambia este texto" o "ahora muestra este botón". Solo cambias el estado y React hace el resto.

💡 En esta guía vas a aprender a hacer que tus componentes sean interactivos y dinámicos, como las apps que usas todos los días.

Objetivos de aprendizaje

  • 🎯 Comprender qué es el estado (state) y por qué es la forma correcta de guardar información que cambia en pantalla.
  • 👆 Reconocer cómo React maneja eventos como clics, escritura en inputs y envío de formularios.
  • 🪝 Utilizar el hook useState para crear y actualizar estado en componentes funcionales de React.
  • ⚡ Aplicar el hook useEffect para ejecutar código en momentos específicos como cuando el componente aparece o cuando cambia algún valor.
  • 🔧 Identificar y evitar errores comunes al trabajar con estado y efectos en React.

Un evento es cualquier acción que hace el usuario en tu app:

👆 onClick: Hacer clic en un botón

⌨️ onChange: Escribir en un input

📝 onSubmit: Enviar un formulario

👆 onMouseEnter: Pasar el mouse sobre algo

📺 Ejemplo de la vida real:

Cuando presionas el botón de "play" en YouTube, ocurre un evento onClick que le dice al video que empiece a reproducirse.

El estado es la "memoria" de tu componente. Es información que puede cambiar con el tiempo y que, cuando cambia, hace que React actualice la pantalla automáticamente.

Ejemplos de estado en apps que conoces:

  • 👍 Número de likes en Instagram
  • 🛍️ Productos en tu carrito de compras
  • 💬 Texto que estás escribiendo en WhatsApp
  • ✅ Si una tarea está completada o no

💡 Regla de oro:

Si algo cambia en tu app, probablemente deba estar en el estado.

❌ ¿Qué NO debe ir en el estado?

  • Información que nunca cambia (usa constantes)
  • Cálculos que puedes hacer con el estado existente
  • Props que recibes del componente padre

❓ Pregunta rápida:

Los hooks son funciones especiales de React que te dan superpoderes. El más importante es useState.

📦 Analogía:

useState es como una caja con dos cosas:

  1. El valor actual (lo que hay en la caja ahora)
  2. Una función para cambiarlo (la única forma de meter algo nuevo en la caja)

Sintaxis:

const [valor, setValor] = useState(valorInicial);

⚠️ Error común:

// ❌ MAL - Esto NO actualiza la pantalla
let contador = 0;
contador = contador + 1;

// ✅ BIEN - Esto SÍ actualiza la pantalla
const [contador, setContador] = useState(0);
setContador(contador + 1);

Cuando quieres que un input muestre lo que el usuario escribe, debes "controlarlo" con estado.

🔍 ¿Cómo funciona?

  • value={`{busqueda}`}: El input muestra lo que hay en el estado
  • onChange: Cada vez que escribes, actualiza el estado
  • e.target.value: El texto que acabas de escribir

✏️ Completa el código de input controlado:

useEffect te permite hacer cosas en momentos especiales:

  • Cuando el componente aparece por primera vez
  • Cuando cambia algún valor específico
  • Cuando el componente desaparece

⏰ Analogía:

Es como programar alarmas en tu celular:

  • "Cuando me despierte, revisar el clima" (cuando el componente aparece)
  • "Cada vez que llegue un mensaje, sonar" (cuando cambia algo)
  • "Cuando me vaya a dormir, apagar notificaciones" (cuando el componente desaparece)

Sintaxis:

useEffect(() => {
  // Código que quieres ejecutar
}, [dependencias]);

🔀 Ordena los pasos de useEffect:

1. Modificar el estado directamente
// ❌ MAL
const [usuario, setUsuario] = useState({ nombre: 'Ana' });
usuario.nombre = 'Carlos'; // Esto NO funciona

// ✅ BIEN
setUsuario({ nombre: 'Carlos' });
2. Olvidar el array de dependencias en useEffect
// ❌ MAL - Se ejecuta infinitamente
useEffect(() => {
  setContador(contador + 1);
}); // Sin array de dependencias

// ✅ BIEN
useEffect(() => {
  setContador(contador + 1);
}, []); // Solo al inicio
3. No manejar estados de carga

Actividades Propuestas

🗳️ Actividad 1: Laboratorio - Sistema de votación para el curso

📖 Historia:

Tu profesor quiere hacer una votación digital para elegir el destino de la excursión del curso. Te pidió crear una app simple donde los estudiantes puedan votar.

Tu misión:

  1. Crea un componente SistemaVotacion con tres opciones: Playa, Montaña, Parque de diversiones
  2. Cada opción debe tener un botón para votar y un contador de votos (usa useState)
  3. Agrega un botón "Reiniciar votación" que ponga todos los contadores en 0
  4. Bonus: Muestra cuál opción va ganando

📸 Producto esperado:

Una app interactiva donde cada clic actualiza los contadores en tiempo real.

📧 Actividad 2: Formulario de contacto con validación

📖 Historia:

Tu primo de la papelería ahora quiere que los clientes puedan enviar mensajes desde la web. Necesita un formulario que valide los datos antes de enviarlos.

Tu misión:

  1. Crea un componente FormularioContacto con campos para: Nombre (mínimo 3 caracteres), Email (debe contener @), Mensaje (mínimo 10 caracteres)
  2. Usa useState para cada campo
  3. Muestra mensajes de error si el nombre es muy corto, el email no tiene @, o el mensaje es muy corto
  4. Solo permite enviar si todo está correcto

📸 Producto esperado:

Un formulario que valida en tiempo real y muestra mensajes de ayuda al usuario.

✅ Actividad 3: Lista de tareas interactiva

📖 Historia:

Necesitas organizar tus tareas del colegio y quieres una app que te ayude a no olvidar nada.

Tu misión:

  1. Crea un componente ListaTareas con un input para escribir nuevas tareas, un botón "Agregar tarea", y una lista que muestre todas las tareas
  2. Usa useState para el texto del input y el array de tareas
  3. Cada tarea debe tener un botón "Eliminar"
  4. Bonus: Agrega un contador que muestre "Tienes X tareas pendientes"

📸 Producto esperado:

Una app funcional de to-do list donde puedes agregar y eliminar tareas.

🔍 Actividad 4: Buscador de productos

📖 Historia:

Tu primo quiere agregar un buscador a su catálogo para que los clientes encuentren productos rápidamente.

Tu misión:

  1. Crea un componente BuscadorProductos con un array de productos (al menos 10), un input de búsqueda, y una lista filtrada de productos
  2. Usa useState para el texto de búsqueda y los productos
  3. Filtra los productos en tiempo real mientras el usuario escribe
  4. Muestra "No se encontraron productos" si no hay resultados

📸 Producto esperado:

Un buscador que filtra instantáneamente, como el de Amazon o MercadoLibre.

🌐 Actividad 5: Cargando datos de una API

📖 Historia:

Es hora de conectar tu app con datos reales de internet. Vamos a usar una API pública para practicar.

Tu misión:

  1. Crea un componente ListaPokemon que use useEffect para cargar datos al inicio
  2. Haz fetch a: https://pokeapi.co/api/v2/pokemon?limit=20
  3. Muestra "Cargando..." mientras espera
  4. Usa useState para el array de pokemon y el estado de carga
  5. Bonus: Agrega manejo de errores por si falla la conexión

💡 Pista:

La respuesta de la API tiene esta estructura:

{
  "results": [
    { "name": "bulbasaur" },
    { "name": "ivysaur" }
  ]
}

📸 Producto esperado:

Una app que carga datos reales de internet, como las apps profesionales.
