Introducción a REACT y visualización de datos: componentes, JSX, listas/tablas

Imagina que tienes que hacer 30 tarjetas de cumpleaños para tus compañeros de clase. ¿Harías cada una completamente diferente desde cero? Probablemente no. Lo más inteligente sería crear una plantilla con el diseño base y luego solo cambiar el nombre y la fecha en cada una.

React funciona exactamente así, pero para páginas web. En lugar de escribir el código HTML de cada producto, cada comentario o cada publicación por separado, creas una "plantilla" (llamada componente) y React la repite automáticamente con diferentes datos.

Piensa en Instagram:

  • Cada publicación se ve similar: foto, nombre de usuario, likes, comentarios
  • Pero los datos cambian: diferentes fotos, diferentes usuarios
  • Instagram no tiene el código de cada publicación escrito por separado
  • Tiene UN componente "Publicación" que se repite miles de veces

💡 React es crear piezas reutilizables que hacen tu vida más fácil. Aprenderás a pensar en componentes, JSX y cómo mostrar listas de datos de forma eficiente.

Objetivos de aprendizaje

  • 🧩 Comprender la idea de componentes en React y cómo esta forma de pensar te ayuda a construir pantallas ordenadas y reutilizables.
  • 📝 Explicar qué es JSX y por qué React divide la interfaz en piezas pequeñas en lugar de trabajar con HTML tradicional.
  • 🔄 Renderizar listas de datos usando el método .map() y entender la importancia de la prop key para optimizar el rendimiento.
  • 📊 Decidir cuándo usar tarjetas/listas vs tablas para presentar información según el tipo de datos y la experiencia de usuario deseada.
  • ⚙️ Configurar un proyecto React con Vite y comprender la estructura básica de archivos y carpetas de una aplicación React moderna.

React es una herramienta de JavaScript creada por Facebook en 2013 que te ayuda a construir páginas web interactivas de forma más fácil y organizada.

Analogía simple:

Imagina que estás construyendo con bloques de LEGO:

  • Sin React: Tienes que pegar cada pieza individualmente y si quieres cambiar algo, debes desarmar todo
  • Con React: Creas "módulos" (componentes) que puedes conectar, desconectar y reutilizar fácilmente

¿Por qué tantas empresas usan React?

🔧 Es como armar con piezas: Construyes una vez, usas muchas veces

⚡ Es rápido: Solo actualiza lo que cambió, no toda la página

📚 Es popular: Hay muchos tutoriales y ayuda disponible

🌟 Apps conocidas: Instagram, WhatsApp Web, Netflix

🎯 Empareja el concepto con su descripción:

Hay varias formas de crear un proyecto React, pero la más moderna y rápida es usando Vite (pronunciado "vit", como "rápido" en francés).

¿Por qué Vite y no Create React App?

  • ⚡ Mucho más rápido (inicia en milisegundos)
  • 🔥 Hot Module Replacement instantáneo
  • 📦 Configuración mínima
  • 🎯 Recomendado por la documentación oficial de React

JSX es una forma de escribir código que parece HTML pero está dentro de JavaScript. Es como si HTML y JavaScript tuvieran un hijo.

Piénsalo así:

  • Normalmente en HTML escribes: <h1>Hola</h1>
  • En JavaScript puro tendrías que escribir: document.createElement('h1') y muchas líneas más
  • Con JSX en React escribes: <h1>Hola</h1> ¡pero dentro de JavaScript! 🎉

Reglas importantes de JSX:

1. Un solo elemento raíz

Debes envolver todo en un elemento padre o usar Fragment <></>

2. className en lugar de class

Porque class es palabra reservada en JavaScript

3. Cerrar todas las etiquetas

Incluso las que en HTML no lo requieren: <img />, <input />

4. Expresiones JavaScript entre llaves

Puedes insertar cualquier expresión JS válida: {`{nombre}`}, {`{edad * 2}`}

✏️ Completa el código JSX correcto:

React es una librería para construir interfaces. Su idea principal es que una pantalla no se trabaja como un bloque gigante, sino como una suma de piezas pequeñas llamadas componentes.

Tipos de componentes:

Componentes de función (recomendado)
function Saludo() {
  return <h1>¡Hola!</h1>;
}
Componentes de clase (antiguo)
class Saludo extends React.Component {
  render() {
    return <h1>¡Hola!</h1>;
  }
}

Props: Pasando datos a componentes

Las props (propiedades) son la forma de pasar datos de un componente padre a un componente hijo. Son como los parámetros de una función.

🎯 Empareja el tipo de dato con cómo se pasa en props:

Cuando los datos vienen en forma de lista, lo normal es recorrerlos con .map() y renderizar un componente por cada elemento.

⚠️ La prop key es obligatoria

React necesita identificar cada elemento de la lista de forma única para optimizar las actualizaciones.

  • Ayuda a React a identificar qué elementos cambiaron
  • Mejora el rendimiento al actualizar la lista
  • Previene bugs cuando se reordenan elementos

❓ Pregunta rápida:

¿Cuándo usar cada una?

✅ Usa tarjetas/listas cuando:
  • Quieres mostrar información visual (imágenes, descripciones largas)
  • Los datos no necesitan compararse directamente
  • Diseño responsive es importante
  • Ejemplos: productos, posts de blog, perfiles de usuario
📊 Usa tablas cuando:
  • Necesitas comparar datos entre filas
  • Los datos son principalmente numéricos o textuales cortos
  • Quieres ordenar por columnas
  • Ejemplos: reportes, inventarios, transacciones

📝 Actividad 1: Laboratorio - Tu primera tienda online

📖 Historia:

Tu primo acaba de abrir una papelería cerca del colegio y te pidió ayuda para crear un catálogo digital de sus productos. Quiere que los estudiantes puedan ver qué vende desde sus celulares. ¡Es tu oportunidad de usar React!

Tu misión:

  1. Crea un componente TarjetaProducto que muestre:
    • Nombre del producto
    • Precio
    • Stock disponible
  2. Crea un componente ListaProductos con al menos 5 productos de papelería (cuadernos, lápices, borradores, etc.)
  3. Bonus: Agrega una etiqueta "🔥 En oferta" cuando el precio sea menor a 3000 pesos para atraer más clientes

💡 Ejemplo de solución:

function TarjetaProducto({ producto }) {
  return (
    <article className="tarjeta">
      <h3>{producto.nombre}</h3>
      <p>Precio: ${producto.precio}</p>
      <p>Stock: {producto.stock}</p>
      {producto.precio < 3000 && <span>🔥 En oferta</span>}
    </article>
  );
}

function ListaProductos() {
  const productos = [
    { id: 1, nombre: "Cuaderno", precio: 2500, stock: 15 },
    { id: 2, nombre: "Lápiz", precio: 800, stock: 50 },
    { id: 3, nombre: "Borrador", precio: 500, stock: 30 },
    { id: 4, nombre: "Regla", precio: 1200, stock: 20 },
    { id: 5, nombre: "Tijeras", precio: 3500, stock: 10 }
  ];

  return (
    <div>
      <h1>Papelería El Estudiante</h1>
      {productos.map(p => (
        <TarjetaProducto key={p.id} producto={p} />
      ))}
    </div>
  );
}

📸 Producto esperado:

Una pantalla que muestre todos los productos como tarjetas, igual que en MercadoLibre o Amazon.

📊 Actividad 3: Sistema de inventario - Modo administrador

📖 Historia:

Tu primo quedó tan contento con el catálogo que ahora quiere una vista especial para él, donde pueda ver rápidamente qué productos se están agotando y necesita reponer.

Tu misión:

  1. Crea un componente TablaProductos que muestre los productos en formato tabla con columnas:
    • Nombre
    • Precio
    • Stock
    • Estado
  2. En la columna "Estado" debe aparecer:
    • "⚠️ Poco stock" si quedan menos de 10 unidades
    • "✅ Disponible" si hay 10 o más

💡 Ejemplo de solución:

function TablaProductos() {
  const productos = [
    { id: 1, nombre: "Cuaderno", precio: 2500, stock: 5 },
    { id: 2, nombre: "Lápiz", precio: 800, stock: 50 }
  ];

  return (
    <table>
      <thead>
        <tr>
          <th>Nombre</th>
          <th>Precio</th>
          <th>Stock</th>
          <th>Estado</th>
        </tr>
      </thead>
      <tbody>
        {productos.map(p => (
          <tr key={p.id}>
            <td>{p.nombre}</td>
            <td>${p.precio}</td>
            <td>{p.stock}</td>
            <td>
              {p.stock < 10 ? '⚠️ Poco stock' : '✅ Disponible'}
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}

📸 Producto esperado:

Una tabla ordenada que permita comparar rápidamente el inventario, como las que usan los supermercados.

🔧 Actividad 4: Componentes flexibles - Dos vistas, un componente

📖 Historia:

Tu primo te dice: "Me encanta la tarjeta de producto, pero cuando la comparto en redes sociales no quiero que se vea el stock, solo el nombre y precio. ¿Puedes hacer que funcione para ambos casos?"

Tu misión:

  1. Modifica TarjetaProducto para que reciba una prop llamada mostrarStock
  2. Implementa la lógica:
    • Si mostrarStock={true} → muestra el stock
    • Si mostrarStock={false} → oculta el stock

💡 Ejemplo de uso:

function TarjetaProducto({ producto, mostrarStock = true }) {
  return (
    <article>
      <h3>{producto.nombre}</h3>
      <p>Precio: ${producto.precio}</p>
      {mostrarStock && <p>Stock: {producto.stock}</p>}
    </article>
  );
}

// Para el catálogo
<TarjetaProducto producto={producto} mostrarStock={true} />

// Para redes sociales
<TarjetaProducto producto={producto} mostrarStock={false} />

📸 Producto esperado:

Un componente inteligente que se adapta según lo necesites. ¡Esto es reutilización real!

✨ Actividad 5: Mini-reto - El toque profesional

📖 Historia:

Tu primo vio la tienda online de un competidor y notó que tiene un encabezado bonito con el nombre de la tienda y un eslogan. Te pregunta: "¿Podemos agregar eso a nuestra página?"

Tu misión:

  1. Crea un componente Header que reciba dos props:
    • titulo: El nombre de la tienda
    • subtitulo: Un eslogan o descripción
  2. Usa este componente dentro de ListaProductos para que la página tenga un encabezado profesional

💡 Ejemplo:

function Header({ titulo, subtitulo }) {
  return (
    <header style={{ 
      backgroundColor: '#4CAF50', 
      color: 'white', 
      padding: '20px',
      textAlign: 'center'
    }}>
      <h1>{titulo}</h1>
      <p>{subtitulo}</p>
    </header>
  );
}

// Uso
<Header 
  titulo="Papelería El Estudiante" 
  subtitulo="Todo lo que necesitas para triunfar en el cole" 
/>

📸 Producto esperado:

Una página completa con encabezado reutilizable que puedes usar en diferentes secciones de la tienda.
