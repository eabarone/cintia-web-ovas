Consumo de APIs desde React

Imagina que estás en un restaurante:

  • 👤 Tú (el cliente) = React / Frontend
  • 📝 El mesero = API
  • 👨‍🍳 La cocina = Backend / Base de datos

🍽️ El flujo del restaurante:

  1. Tú le pides al mesero: "Quiero ver el menú" (GET request)
  2. El mesero va a la cocina y trae el menú (respuesta de la API)
  3. Tú ordenas: "Quiero una hamburguesa" (POST request)
  4. La cocina prepara tu orden (el backend procesa)
  5. El mesero te trae tu comida (respuesta exitosa)

Eso es exactamente lo que hace una API: conecta lo que el usuario ve (React) con donde se guardan los datos (backend).

¿Por qué es importante?

Hasta ahora, tus apps de React tenían datos "inventados" dentro del código. Pero las apps reales traen datos de servidores:

  • Instagram trae tus fotos desde sus servidores
  • Netflix trae las películas desde su base de datos
  • WhatsApp envía tus mensajes a otros usuarios

En este tema vas a aprender a hacer que tu React se comunique con APIs reales. ¡Es el momento de hacer apps de verdad! 🚀

Objetivos de aprendizaje

  • 🌐 Conectar React con APIs usando fetch() para traer datos de un servidor y mostrarlos en tu interfaz.
  • ⏳ Manejar estados de carga mostrando "Cargando...", mensajes de error y datos exitosos de forma profesional.
  • 📤 Enviar datos con POST para crear nuevos recursos en el backend desde formularios de React.
  • 🔍 Entender códigos HTTP para interpretar respuestas del servidor (200, 404, 500) y manejarlas correctamente.
  • 🚀 Crear apps full-stack conectando tu frontend de React con tu backend de Node.js para hacer aplicaciones reales.

Cuando React se conecta a una API, sigue estos pasos:

1️⃣ Estado inicial: Cargando...

const [cargando, setCargando] = useState(true);

La app muestra "Cargando..." para que el usuario sepa que algo está pasando.

2️⃣ Hacer la petición (fetch)

const respuesta = await fetch('https://api.ejemplo.com/datos');

React le pide datos al servidor.

3️⃣ Procesar la respuesta

const datos = await respuesta.json();

Convierte la respuesta en un formato que JavaScript entiende.

4️⃣ Guardar en el estado

setDatos(datos);
setCargando(false);

Guarda los datos y quita el mensaje de "Cargando...".

5️⃣ Mostrar en pantalla

{datos.map(item => 
{item.nombre}
)}

React renderiza los datos automáticamente.

🔀 Ordena los pasos del flujo:

fetch() es la función de JavaScript que te permite hacer peticiones HTTP (hablar con APIs).

📞 Analogía:

fetch() es como hacer una llamada telefónica:

  1. Marcas el número (URL de la API)
  2. Esperas que contesten (promesa pendiente)
  3. Hablas y recibes información (respuesta)
  4. Cuelgas (promesa resuelta)

Sintaxis con async/await (moderna y fácil):

try {
  const respuesta = await fetch('URL');
  const datos = await respuesta.json();
  console.log(datos);
} catch (error) {
  console.error(error);
}

Cuando hablas con una API, le dices qué acción quieres hacer:

GET 📥

"Dame información" (leer)

Ejemplo: Ver la lista de productos

POST 📤

"Crea algo nuevo" (crear)

Ejemplo: Agregar producto al carrito

PUT/PATCH ✏️

"Actualiza esto" (modificar)

Ejemplo: Cambiar tu nombre de usuario

DELETE 🗑️

"Elimina esto" (borrar)

Ejemplo: Eliminar una foto

❓ Pregunta rápida:

Cuando haces una petición, el servidor responde con un código que te dice qué pasó:

2xx - Éxito ✅

  • 200 OK: Todo bien, aquí están tus datos
  • 201 Created: Se creó exitosamente
  • 204 No Content: Se eliminó exitosamente

4xx - Error del cliente ⚠️

  • 400 Bad Request: Enviaste datos incorrectos
  • 401 Unauthorized: Necesitas iniciar sesión
  • 404 Not Found: No existe lo que buscas

5xx - Error del servidor 🔥

  • 500 Internal Server Error: El servidor tiene un problema
  • 503 Service Unavailable: El servidor está caído

Cómo verificar el código:

const respuesta = await fetch('URL');

if (respuesta.ok) {
  // Código 2xx - Todo bien
  const datos = await respuesta.json();
} else {
  // Código 4xx o 5xx - Hubo un error
  throw new Error(`Error ${respuesta.status}`);
}

🎯 Empareja el código con su significado:

Cuando quieres crear algo nuevo (un comentario, un producto, un usuario), usas POST.

📝 Puntos importantes:

  1. method: 'POST' - Le dice que quieres crear algo
  2. headers - Le dice que estás enviando JSON
  3. body: JSON.stringify() - Convierte tu objeto a texto JSON
  4. disabled={enviando} - Evita envíos múltiples

✨ Actividad 3: Mejora la experiencia del usuario (UX)

📖 Historia:

Un compañero probó tu app y te dijo: "Cuando creo una tarea, no sé si se está enviando o si ya terminó. Además, puedo hacer clic varias veces en el botón y se envía múltiples veces".

Tu misión:

  1. Agrega un estado enviando para el formulario:

  2. Desactiva el botón mientras se envía:

  3. Cambia el texto del botón según el estado:

❌ Actividad 4: Manejo profesional de errores

📖 Historia:

Tu profesor intentó crear una tarea sin título y la app se rompió completamente. Te pide que manejes los errores de forma profesional.

Tu misión:

  1. Modifica tu backend para que valide el título:
    • Si está vacío, responde con error 400 y mensaje: { "error": "El título es obligatorio" }
  2. En React, captura ese error y muéstralo al usuario: 
    const errorData = await res.json();
setError(errorData.error || 'Error desconocido');
  3. Agrega validación en el frontend también:
    • No permitas enviar el formulario si el título está vacío
    • Muestra un mensaje de ayuda
  4. Bonus: Usa diferentes colores para mensajes de error (rojo) y éxito (verde)

📸 Producto esperado:

Una app robusta que maneja errores sin romperse y guía al usuario para corregirlos.

Recursos para profundizar

APIs públicas gratuitas para practicar consumo de datos desde React:

  • JSONPlaceholder - API de prueba

    API falsa gratuita para testing y prototipos. Incluye posts, comentarios, usuarios, fotos y más.

    jsonplaceholder.typicode.com

  • PokeAPI - Datos de Pokémon

    API completa con información de todos los Pokémon, habilidades, movimientos y más. Perfecta para practicar.

    pokeapi.co

  • Rick and Morty API

    API con personajes, episodios y locaciones de la serie. Excelente para practicar listas y filtros.

    rickandmortyapi.com

  • OpenWeatherMap - API de Clima

    Datos meteorológicos en tiempo real. Requiere registro gratuito para obtener API key.

    openweathermap.org/api

💡 Consejo:

Estas APIs son gratuitas y no requieren autenticación (excepto OpenWeatherMap). Son perfectas para practicar fetch(), manejo de estados y mostrar datos en React.
