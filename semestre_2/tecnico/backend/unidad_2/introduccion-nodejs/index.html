Introducción a Node.js

Hasta ahora, cuando pensabas en JavaScript seguramente lo imaginabas dentro del navegador: botones que reaccionan, animaciones, formularios, páginas que cambian sin recargar.

Pero las aplicaciones reales tienen otra parte que casi nunca se ve: un lugar donde se guardan usuarios, contraseñas, publicaciones, productos y cualquier dato importante. Esa parte no vive en el navegador; vive en un servidor. Y es justo ahí donde aparece Node.js 💻.

Con Node.js, JavaScript deja de ser "solo del frontend" y se convierte también en un lenguaje para construir el backend. Esto te permite crear programas que reciben peticiones, procesan información, se conectan a bases de datos y responden con datos listos para que tu interfaz los muestre.

✨ En otras palabras: estás a un paso de construir un sistema completo, donde tú controlas tanto la pantalla como lo que sucede detrás 🚀.

Objetivos de aprendizaje

  • 🧠 Comprender qué es Node.js y por qué es una herramienta tan usada para crear el "lado servidor" de una aplicación web.
  • 💡 Explicar cómo Node.js permite ejecutar JavaScript fuera del navegador y qué tipo de tareas realiza un backend.
  • 🔧 Entender el modelo asíncrono y no bloqueante de Node.js basado en el Event Loop.
  • 📦 Conocer el ecosistema de npm y cómo gestionar paquetes en un proyecto Node.js.
  • 🚀 Organizar un proyecto inicial para empezar a construir una API siguiendo buenas prácticas desde el inicio.

Node.js es un entorno de ejecución que permite ejecutar JavaScript fuera del navegador. Tradicionalmente, JavaScript solo funcionaba en navegadores web, pero Node.js cambió eso al permitir que JavaScript se ejecute como un programa en un servidor o en tu computadora local.

Fue creado en 2009 por Ryan Dahl, utilizando el motor V8 de Google Chrome (el mismo que ejecuta JavaScript en el navegador). La gran diferencia es que Node.js añade capacidades para interactuar con el sistema operativo: leer archivos, crear servidores, conectarse a bases de datos, y mucho más.

¿Por qué es tan popular Node.js?

  • Un solo lenguaje: Puedes usar JavaScript tanto en el frontend como en el backend, lo que simplifica el desarrollo.
  • Asíncrono y eficiente: Node.js maneja múltiples peticiones al mismo tiempo sin bloquear el servidor.
  • Ecosistema enorme: npm (Node Package Manager) tiene millones de paquetes listos para usar.
  • Comunidad activa: Hay mucha documentación, tutoriales y soporte disponible.

Node.js funciona de manera diferente a otros lenguajes de backend como PHP o Java. Utiliza un modelo asíncrono y no bloqueante basado en el Event Loop.

¿Qué significa esto? Imagina un restaurante:

  • Modelo tradicional (bloqueante): Un mesero atiende a un cliente, espera que termine de comer, cobra, y solo entonces atiende al siguiente. Si hay 10 clientes, el último espera mucho tiempo.
  • Modelo Node.js (no bloqueante): Un mesero toma la orden del primer cliente, la pasa a la cocina, y mientras se prepara la comida, atiende a otros clientes. Cuando la comida está lista, la entrega. Así puede manejar muchos clientes al mismo tiempo.

En Node.js, cuando haces una operación que toma tiempo (como leer un archivo o consultar una base de datos), el servidor no se queda esperando. En su lugar, continúa procesando otras peticiones y cuando la operación termina, ejecuta una función de callback.

Node.js organiza el código en módulos. Un módulo es simplemente un archivo JavaScript que exporta funciones, objetos o variables para que otros archivos puedan usarlos.

Existen tres tipos de módulos:

  • Módulos nativos: Vienen incluidos con Node.js (como fs para archivos, http para servidores, path para rutas).
  • Módulos de terceros: Se instalan con npm (como express, mongoose, dotenv).
  • Módulos propios: Los que tú creas para organizar tu código.

Ejemplo de módulo propio

Simulación de funciones de módulo:

function sumar(a, b) {
  return a + b;
}

function restar(a, b) {
  return a - b;
}

function multiplicar(a, b) {
  return a * b;
}

console.log(sumar(5, 3));
console.log(restar(10, 4));
console.log(multiplicar(6, 7));

El archivo package.json

Este archivo es el corazón de tu proyecto Node.js. Contiene información sobre el proyecto y sus dependencias.

{
  "name": "mi-api",
  "version": "1.0.0",
  "type": "module",
  "description": "API de tareas",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.1"
  }
}

¿Qué es type: "module"?

Esto le dice a Node.js que uses la sintaxis moderna de módulos ES6 (import/export) en lugar de la antigua (require/module.exports).

Cuando creas una API con Node.js, normalmente estás construyendo un servidor que expone rutas (endpoints) como /usuarios o /tareas. Cada vez que el frontend hace una petición, el backend:

  1. Recibe la solicitud
  2. Valida los datos
  3. Procesa la lógica (consultar BD, calcular algo, etc.)
  4. Devuelve una respuesta con un código de estado y datos (generalmente JSON)

Express es el framework más popular para crear APIs en Node.js. Simplifica enormemente el proceso de crear un servidor y manejar rutas.

Aunque cada proyecto puede organizarse distinto, una estructura clara te ayuda a mantener el código ordenado y escalable. Aquí una estructura recomendada para empezar: 

mi-proyecto/
├── src/
│   ├── routes/          # Definición de rutas/endpoints
│   │   └── tareas.js
│   ├── controllers/     # Lógica de negocio
│   │   └── tareasController.js
│   ├── models/          # Modelos de datos
│   │   └── Tarea.js
│   ├── middlewares/     # Funciones intermedias
│   │   └── validacion.js
│   └── db/              # Configuración de base de datos
│       └── conexion.js
├── index.js             # Punto de entrada
├── package.json         # Dependencias y scripts
└── .env                 # Variables de entorno

Separación de responsabilidades

  • Routes: Define qué endpoints existen y qué método HTTP usan
  • Controllers: Contiene la lógica de lo que hace cada endpoint
  • Models: Define la estructura de los datos
  • Middlewares: Funciones que se ejecutan antes de llegar al controlador (validaciones, autenticación, etc.)

🔀 Ordena el flujo de una petición API:

Arrastra las líneas para ordenar correctamente el flujo de una petición en Express:

Si desde el comienzo aplicas buenas prácticas, tu API será más fácil de mantener y escalar:

  1. Nombres de rutas con sustantivos: Usa /usuarios, /tareas, /productos (no /obtenerUsuarios o /crearTarea)
  2. Usa los métodos HTTP correctamente: GET para leer, POST para crear, PUT/PATCH para actualizar, DELETE para eliminar
  3. Códigos de estado apropiados: 200 (OK), 201 (Creado), 400 (Error del cliente), 404 (No encontrado), 500 (Error del servidor)
  4. Respuestas consistentes: Siempre devuelve JSON con la misma estructura
  5. Manejo de errores: Nunca dejes que el servidor se caiga sin un mensaje claro
  6. Variables de entorno: Guarda información sensible (contraseñas, API keys) en archivos .env
  7. Validación de datos: Siempre valida lo que recibes del cliente

Actividades propuestas

A continuación encontrarás cinco actividades prácticas para aplicar lo aprendido sobre Node.js. Estas actividades te ayudarán a consolidar los conceptos fundamentales del desarrollo backend.

🚀 Actividad 1: Laboratorio (tu primer servidor)

Vas a levantar tu primer servidor en Node.js.

  1. Crea una carpeta nueva para el proyecto
  2. Inicializa Node y crea un archivo index.js
  3. Instala Express
  4. Copia el código del ejemplo del desarrollo
  5. Ejecuta el servidor y prueba en el navegador: http://localhost:3000/ y http://localhost:3000/tareas

📝 Producto esperado: Un servidor funcionando y un pequeño reporte (1 párrafo) explicando qué devuelve cada endpoint.

💬 Actividad 2: Mini-reto

Agrega un endpoint nuevo GET /saludo que devuelva un JSON con esta forma: 

{ "mensaje": "Hola, soy una API" }

📝 Producto esperado: Prueba la ruta en el navegador y toma una captura.

🔄 Actividad 3: Middleware básico

Agrega app.use(express.json()) (si no lo tienes) y crea un endpoint POST /eco.

La respuesta debe devolver exactamente lo que le enviaste en JSON.

📝 Producto esperado: Prueba en Postman con un JSON cualquiera.

📁 Actividad 4: Organización del proyecto

Organiza tu proyecto en carpetas (aunque sea básico).

  1. Crea una carpeta routes
  2. Mueve la ruta /tareas a un archivo separado y expórtala

📝 Producto esperado: El servidor sigue funcionando igual.

🔍 Actividad 5: Mini-reto (endpoint con parámetro)

Crea un endpoint GET /tareas/:id.

Si el id no existe, devuelve 404.

📝 Producto esperado: Prueba con un id válido y uno inválido.
